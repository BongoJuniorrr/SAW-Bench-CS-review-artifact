# `annotation/` — File Inventory and Reading Guide

Scope of the labeling effort: **403 warnings, 3,099 candidate snippets, 2 released labeling passes** (A and B). Both passes are merged to construct the released benchmark file `data/saw_bench_cs.jsonl`.

---

## A–B Inter-Rater Agreement Summary

Pairwise snippet-level agreement between the two released passes (April 2026):

| Metric | A–B |
|--------|-----|
| Observed agreement | 62.3% (1,931 / 3,099) |
| Cohen's κ (3-class) | **0.3262** |
| PABAK | 0.4347 |
| Essential agreement | 77.6% |
| Helpful agreement | 62.0% |
| Irrelevant agreement | 4.9% |

**Key observations:**
- **Essential labels have the strongest A/B support** (77.6% agreement): the core construct is more stable than the other two classes.
- **Helpful labels show moderate agreement** (62.0%): supporting context is less consistently identified.
- **Irrelevant labels have near-zero agreement** (4.9%): the "irrelevant" category is a construct-validity limitation in this release.

## Inter-Rater Agreement & Construct Validity Findings

### Summary

The two-pass labeling effort reveals:

1. **Essential labels have the strongest A/B support** (77.6% A–B agreement)
   → The core construct is more stable than helpful/irrelevant labeling.

2. **Helpful labels show moderate agreement** (62.0% A–B)
   → Supporting context is less consistently identified but still meaningful.

3. **Irrelevant labels are least stable** (4.9% A–B agreement)
   → This is the core construct validity issue: what counts as "irrelevant" is highly subjective.

### Key Insight: The Irrelevant Problem
- **A vs B**: Only 4.9% agreement on irrelevant labels despite 77.6% on essential
- **Pattern**: A marks things helpful; B marks the same things irrelevant
- **Consequence**: If RQ4 evaluation penalizes including "irrelevant" snippets, it may penalize helpful context
- **Evidence**: the A/B confusion matrix has many A-helpful/B-irrelevant and A-irrelevant/B-helpful disagreements.

### Recommendations for Interpretation
1. **Use A–B for published benchmark**, but acknowledge 4.9% irrelevant disagreement
2. **Weight labels by confidence**: Essential (high) > Helpful (medium) > Irrelevant (low)
3. **For RQ4**: Don't penalize LLMs for including snippets marked irrelevant; disagreement data shows these are often helpful
4. **For future**: Multi-rater consensus on irrelevant category with clearer operational definition

## How to reproduce the numbers in the paper

The agreement statistics in §4.2 and Appendix A are recomputed from `annotator_passes.jsonl` by `scripts/compute_annotation_agreement.py` — that script writes `annotator_agreement.json`. The pairwise audit JSON in group 3 is produced by `scripts/compare_annotation_passes.py`. The RQ4 calibration log in group 4 is produced by `scripts/generate_rq4_artifacts.py`. Nothing in this folder is hand-edited after generation; deleting any file and re-running the corresponding script reproduces it bit-for-bit.

## What this folder is *not*

It is not a human-subject annotation set. The pass JSONLs do not embed human-rater identities, consent records, or model identifiers. The paper should therefore describe A and B as artifact-labeling passes, not as independent human annotators, unless a separate raw human annotation package is added.
